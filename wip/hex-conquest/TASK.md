# ヘックス制圧 v6 - 大規模改修タスク

## 現在のファイル
`index.html` — 単一ファイルのCanvas APIゲーム（外部ライブラリなし）

## 改修項目（全て必須）

### 1. マップサイズ拡大（マルチレーン化）
- COLS/ROWSをstageに応じて変動させる
  - Stage 1-2: COLS=5, ROWS=9 (1レーン)
  - Stage 3-4: COLS=7, ROWS=9 (2レーン)
  - Stage 5+:  COLS=9, ROWS=9 (3レーン)
- calcGrid()でhexSizeを動的に再計算
- 城の位置もCOLS中央に合わせる

### 2. 配置制限（城の近くのみ）
- canPlace()を変更: 城から2マス以内にのみ配置可能
- 現在: 自軍所有マスの隣接 → 変更: playerCastleからhexDist <= 2

### 3. 移動アニメーション
- 現在: 移動は点線だけ → ユニットアイコンが実際にスライドする
- move animの間、移動中ユニットの座標をfrom→toへ補間して描画
- drawAnims()で移動中ユニットを元の位置から目的地へ動かす

### 4. 逐次行動システム（最重要）
- 現在: 全ユニット同時に攻撃判定 → 「空振り」が発生
- 変更: 1体ずつ処理
  - ユニットAが移動→攻撃→死亡処理（ここでgrid更新）
  - ユニットBが移動→攻撃→死亡処理
  - ...
- 具体的な実装:
  - doPlayerTurn()/doEnemyTurn()を「1体分のアニメキュー生成」に変更
  - processAnims()で1体分が終わったら次の1体を処理
  - ユニットリストを保持し、currentUnitIdxで管理
  - 攻撃前にターゲットの生存確認（空振り防止）

### 5. スキルツリー大幅拡張
- 現在12ノード → 20ノード以上に拡張
- 5ブランチ構成:
  - ⚔️ 攻撃(ATK系、クリティカル)
  - 🛡️ 防御(HP系、反撃、城HP)
  - 📡 展開(配置数、移動力)
  - 🏹 射撃(弓射程、弓ATK)
  - 🧙 特殊(全体バフ、回復、範囲攻撃)
- スキルツリー画面もスクロール or 複数ページ対応

### 6. セーブ/ロード (localStorage)
- 保存する値: stage, score, sp, unlockedSkills, fastMode, soundOn
- `hexconquest_save` キーでJSON保存
- タイトル画面に「続きから」ボタン（セーブがある場合のみ表示）
- ステージクリア時に自動セーブ

### 7. リセットボタン
- タイトル画面に「リセット」ボタン追加
- localStorageを消去して初期状態に

### 8. 速度トグル常時表示
- HUD上部に常に速度ボタンを表示（戦闘中だけでなく）
- 位置: STAGE表示の横あたり
- 配置フェーズ中でもタップで切替可能

### 9. SP/スピーカー被り修正
- 右上レイアウト:
  - SP表示: 右上1行目
  - スピーカーマーク: その下 or 左横にずらす
- 重ならないように十分な間隔を取る

## 技術的制約
- Canvas API のみ（外部ライブラリ禁止）
- 単一HTMLファイル
- devicePixelRatio対応済み（維持すること）
- getBoundingClientRect()でタッチ座標取得（維持すること）
- モバイルファースト: 375x812想定
- タッチターゲット最低48px
- フォント最小14px
- 全てのアニメーションはfastMode対応
